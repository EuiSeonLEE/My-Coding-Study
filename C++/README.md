# C++ 프로그래밍 언어
## 컴퓨터 언어 및 컴파일러의 개념

- 프로그램 == 작업지시서
    - 컴퓨터에게 해야 할 작업의 내용을 알려주는 문서
    - 상식이나 지능이 없기 대문에 구체적으로 일을 지시
- 컴퓨터 언어의 분류
    - 기계어(컴퓨터에게 친화적)
    - 어셈블리어
    - 고급언어(인간에게 친화적) : 컴퓨터가 이해하게 할 번역을 위한 도구 필요(컴파일러)
- **컴파일러**
    - 인간과 컴퓨터 사이의 통역하는 컴파일 과정을 지원하는 도구
    - 고급언어로 작성된 프로그램을 기계어로 번역
    - 해석기는 기계어를 기본연산으로 수행
- **프로그램 개발 과정**
    - 요구사항 분석 → 알고리즘 개발 → 코딩 → 컴파일과 링크 → 실행과 디버깅 → 유지보수
    - editer(에디터)로 test.c(소스파일)을 작성하고
    compiler(컴파일러)로 test.obj(오브젝트 파일)을 만들고
    linker(링커)로 test.exe(실행파일)을 만든다.
    위를 전부 종합적으로 한번에 해주는 것이 **통합 개발 환경(IDE)**이다.
- 오류 수정 및 디버깅
    - 개발자의 설계 및 입력 오류에 따라 컴파일이나 실행 시에 에ㅔ러나 경고가 발생
    - 에러와 경고
        - 에러(error)
            - 심각한 오류
            - 실행 파일 생성 안되므 오류 수정 후 다시 컴파일
        - 경고(warning)
            - 경미한 오류
            - 사용하지 않는 변수 및 기능 작성 안된 경우
            - 개발자가 이를 무시할 경우 도구도 같이 무시하여 처리
            - 심각한 실행 오류 가능성 존재
- 오류와 경고 종류
    - 컴파일 시간 오류
        - 문법적 오타
        - 구현되지 않는 기능 호출
        - 도구에 의해 쉽게 해결 가능함
    - 실행 시간 오류
        - 입력 오류 : 숫자 입력에서 문자
        - 연산 오류 : 0으로 나누기, 값의 범위 넘기기
        - 디버깅과 같은 도구를 이용하여 추적 해결
    - 논리 오류
        - 프로그램 개발 능력에 따른 오류
        - 디버깅과 같은 도구를 이용하여 추적
        - 프로그램 재 설계
- 컴파일 오류
    - 커파일 단계에서 오류가 발생한 경우, 해당 부분 수정 후 개 컴파일 처리
